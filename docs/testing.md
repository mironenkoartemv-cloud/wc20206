# Тестирование Chexx WC 2026 Predictor

## Автоматические проверки

Синтаксическая проверка:

```bash
npm run check
```

Backend API suite:

```bash
CHEXX_DB_FILE=backend/data/test-db.json PORT=8788 ADMIN_TOKEN=test-admin node backend/server.mjs
npm run backend:test
```

API suite покрывает:

- health endpoint;
- статическую раздачу фронта;
- пустой leaderboard;
- отсутствие access token;
- пустое имя игрока;
- невалидный JSON;
- создание игрока;
- получение игрока по token;
- запрет сохранения прогноза без token;
- повторяющиеся команды в группе;
- команда не из своей группы;
- неверное количество третьих мест;
- несовпадение `thirdPlaces.teamId` с третьей командой в группе;
- несовпадение `championTeamId` с победителем финала;
- сохранение валидного прогноза;
- перезапись прогноза без дубля в leaderboard;
- закрытый admin endpoint при `ADMIN_TOKEN`;
- начисление третьих мест по `teamId`, а не по `groupId`;
- идеальный mock-результат на 119 очков;
- неизвестный API route.

## Browser-проверка

Проверялись сценарии:

- блок прогноза скрыт до ввода имени;
- пустое имя не открывает flow;
- валидное имя открывает группы;
- переход к третьим местам требует ровно 8 выборов;
- сохранение недоступно до заполнения всей сетки;
- заполненная сетка сохраняется через backend;
- live-таблица показывает backend-строку и сохраненный прогноз;
- reload восстанавливает игрока, live-экран и сохраненный прогноз по token.

## Найденные и исправленные bugs

1. Backend игнорировал присланный `thirdPlaces.teamId`, если в payload был `thirdGroups`, и пересобирал thirdPlaces сам. Из-за этого нельзя было поймать рассинхрон между третьей командой в группе и payload. Исправлено в `normalizePredictionPayload()` и усилено валидацией.

2. Frontend `API_BASE` был жестко привязан к `localhost:8787`. При запуске backend на другом порту или в deploy-окружении фронт мог ходить не туда. Исправлено: same-origin по умолчанию, dev fallback только для статических dev-портов.

3. После reload с сохраненным token backend возвращал игрока, но `predictionFlow` оставался скрытым, потому что `flowStarted` не восстанавливался. Исправлено в `syncBackendData()`.

4. API suite раньше зависел от состояния `backend/data/test-db.json` между прогонами. Добавлен `POST /api/admin/reset`, тесты теперь сами очищают изолированную базу.
